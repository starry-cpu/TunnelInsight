# TunnelInsight 前端开发指南

## 快速启动

### 1. 启动开发服务器

```bash
cd frontend
npm run dev
```

访问 http://localhost:3000

### 2. 开发模式登录

由于后端 API 尚未部署，请使用**开发模式登录**来测试前端界面：

1. 访问登录页面：http://localhost:3000/login
2. 点击页面底部的 **"开发模式登录（无需后端）"** 按钮
3. 登录成功后，即可访问所有功能页面

## 功能页面导航

登录后，通过左侧导航菜单访问各功能页面：

### 📊 项目概览 (`/dashboard`)
- 查看所有监测项目的统计信息
- 项目健康指数和缺陷统计
- 支持搜索和筛选

### 🔬 缺陷分析 (`/analysis`)
- 上传隧道缺陷图片
- 配置 AI 分析参数（Temperature, Top-K, Top-P, Max Tokens）
- 查看 Markdown 格式的分析结果
- 复制和导出功能

### 🗂️ 隧道管理 (`/tunnels`)
- 查看所有隧道列表
- 创建新隧道
- 编辑隧道信息
- 删除隧道
- 查看隧道详情

### 📜 历史记录 (`/history`)
- 查看所有分析记录
- 按日期、隧道、类型筛选
- 查看详细分析结果
- 重新分析功能
- 删除记录

## 页面功能说明

### 缺陷分析页面

**左侧面板**：
1. **上传图片**：支持拖拽上传，最大 10MB
2. **分析参数**：
   - Temperature (0.1-2.0)：控制输出随机性
   - Top-K (1-100)：保留最高概率的标记数
   - Top-P (0.1-1.0)：核采样参数
   - Max Tokens (100-4096)：最大生成长度
   - Instruction：自定义提示词

**右侧面板**：
- 实时显示分析结果（Markdown 格式）
- 复制按钮：复制结果到剪贴板
- 导出 PDF 按钮（开发中）

### 隧道管理页面

**功能**：
- **新建隧道**：点击右上角"新建隧道"按钮
- **编辑隧道**：点击卡片上的"编辑"按钮
- **删除隧道**：点击卡片上的"删除"按钮
- **查看详情**：点击卡片或"详情"按钮

**隧道卡片信息**：
- 隧道名称和位置
- 健康指数（进度条）
- 状态标签（正常/预警/严重）
- 缺陷总数和全长

### 历史记录页面

**统计卡片**：
- 总记录数
- 今日记录数
- 本周记录数

**筛选功能**：
- 按隧道筛选
- 按缺陷类型筛选
- 按严重程度筛选（低/中/高/严重）
- 按日期范围筛选

**表格功能**：
- 缩略图预览
- 查看详情（点击眼睛图标）
- 重新分析（点击刷新图标）
- 删除记录（点击删除图标）

## 开发提示

### 模拟数据

目前使用模拟数据展示功能：
- 隧道列表使用随机生成的状态和进度
- 历史记录表格在没有实际数据时显示空状态

### API 对接

后端 API 开发完成后，需要：
1. 更新 `src/services/` 中的服务类
2. 移除模拟数据逻辑
3. 对接真实的 API 端点
4. 移除开发模式登录功能

### 样式自定义

- 全局样式变量：`src/assets/styles/variables.css`
- 组件样式：各组件的 `.css` 文件
- 主题颜色：
  - 主色：`#3B82F6` (蓝色)
  - 成功：`#10B981` (绿色)
  - 警告：`#F59E0B` (黄色)
  - 危险：`#EF4444` (红色)

## 故障排查

### 无法访问页面

1. **确认已登录**：使用开发模式登录
2. **检查控制台**：F12 打开浏览器控制台查看错误
3. **刷新页面**：Ctrl+F5 或 Cmd+Shift+R

### 页面空白

1. 打开浏览器控制台查看错误信息
2. 确认开发服务器正在运行
3. 清除浏览器缓存

### 样式问题

1. 确认 `global.css` 已导入
2. 检查 CSS 变量是否正确定义
3. 清除浏览器缓存并刷新
