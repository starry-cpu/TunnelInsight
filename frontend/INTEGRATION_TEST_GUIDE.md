## 🚀 快速测试

### 1. 启动开发服务器

**后端**:
```bash
cd backend
pip install -r requirements.txt
alembic upgrade head  # 确保数据库迁移已运行
uvicorn app.main:app --reload
```

**前端**:
```bash
cd frontend
npm install
npm run dev
```

### 2. 测试流程

#### 方式A: 使用开发模式（推荐）

1. 访问 http://localhost:5173/login
2. 点击"忘记密码?"链接
3. 输入任意邮箱（如：test@example.com）
4. 点击"发送重置链接"
5. **在页面底部查看开发模式的token**
6. 复制token
7. 访问 http://localhost:5173/reset-password/{token}
8. 输入新密码
9. 重置成功后点击"前往登录"
10. 使用新密码登录

#### 方式B: 使用生产模式（需配置SMTP）

1. 配置 `backend/.env`:
   ```bash
   EMAIL_ENABLED=true
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_USER=your-email@gmail.com
   SMTP_PASSWORD=your-app-password
   ```

2. 重启后端服务

3. 访问 http://localhost:5173/forgot-password
4. 输入真实邮箱
5. 检查邮箱收件箱
6. 点击邮件中的重置链接
7. 设置新密码
8. 登录

## 🔧 开发者注意事项

### 环境变量

**开发环境** (`EMAIL_ENABLED=false`):
- 不发送真实邮件
- Token直接显示在页面上
- 可以快速测试流程

**生产环境** (`EMAIL_ENABLED=true`):
- 发送真实邮件
- 不显示token
- 需要配置SMTP

### API响应格式

**成功响应**:
```json
{
  "success": true,
  "data": {
    "message": "If the email exists, a password reset link has been sent",
    "token": "abc123...",  // 仅开发环境
    "debug_mode": true      // 仅开发环境
  }
}
```

**错误响应**:
```json
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Invalid or expired reset token"
  }
}
```

### 前端状态管理

- 使用本地state管理表单状态
- 不需要全局store（简单流程）
- 使用React Router处理路由参数

## 🐛 常见问题

### Q1: 点击重置链接后显示"重置链接无效"

**可能原因**:
- Token已过期（24小时）
- Token已被使用
- Token格式错误

**解决方案**:
- 重新发起忘记密码请求
- 确保完整复制链接

### Q2: 开发模式下没有显示token

**检查清单**:
- [ ] 后端 `.env` 中 `EMAIL_ENABLED=false`
- [ ] 后端服务已重启
- [ ] 浏览器控制台无错误

### Q3: 密码强度一直显示"弱"

**原因**: 密码不满足复杂度要求

**解决方案**:
- 至少8个字符
- 包含大写字母 (A-Z)
- 包含小写字母 (a-z)
- 包含数字 (0-9)
- 建议包含特殊字符 (!@#$%^&*)

### Q4: 生产环境邮件未发送

**检查清单**:
- [ ] `EMAIL_ENABLED=true`
- [ ] SMTP配置正确
- [ ] SMTP密码正确（应用专用密码）
- [ ] 检查后端日志
- [ ] 检查垃圾邮件文件夹
